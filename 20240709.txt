## <생성형 인공지능>

인공지능 모델 VS 프로그래밍

공통점: 개발자가 무엇을 입력으로 넣고 무엇을 출력할지 설계
차이점: 프로그래밍은 입력과 출력의 관계에 대한 알고리즘을 직접 구현, 인공지능 모델은 입력과 출력의 관계를 데이터를 통해 '학습' *(수정됨)*

- 생성형 인공지능(Generative AI) 텍스트, 이미지, 오디오, 비디오 등 새로운 컨텐츠를 생성할 수 있는 인공지능 기술

단계적으로 문제를 해결하는데 성능이 좋다

VAE, GAN, Diffusion, Transformer Decoder

- 프롬프트(Prompt) 생성형 AI에 주어지는 입력 데이터로, 생성형 AI가 출력할 컨텐츠의 내용 및 스타일 등을 안내 하는 역할을 수행
- 이미지 생성 AI 유명 서비스: DALL-E(무료)
- 동영상 생성 AI: Pixverse(무료)
- 텍스트 생성 AI: ChatGPT(OpenAI), Gemini(Google), Claude(코드 생성에 성능좋음)
- 음악 생성 AI: Suno, Udio
- LLM(Large Language Model)
텍스트 생성, 번역, 질문 답변, 요약 등 다양한 언어 관련 작업을 하나의 모델에서 수행하는 대규모 언어 모델

GPT v.3 이 최초

LLaMA3 좋음

토큰은 단어

파라미터는 활용능력있는 뉴런

[LLM 리더보드 2024 (vellum.ai)](https://www.vellum.ai/llm-leaderboard)

- LLM의 원리 입력[프롬프트]과 출력 사이의 조건부 확률 분포를 학습함. 새로운 입력이 주어졌을 때 학습된 조건부 확률 분포를 통해 모든 토큰[단어]별 확률을 예측하고, 그 중 가장 가능성이 높은 토큰을 선택하여 답변함.

## LLM의 한계점

 GPT는 실제로 계산을 할까? 모든 종류의 텍스트 데이터를 기억 및 학습하기 때문에 이 역시 직접 계산하는 것이 아니라 확률적으로 출력해낸다.

통계모델

추론능력 없음

데이터 편향과 데이터 오류

학습 데이터를 벗어나는 입력

- ChatGPT(!=GPT)는 외부 도구를 활용하여 수학적 계산을 처리함

        입력 -> 입력 이해 및 파싱 -> 외부 도구 -> 출력

## LLM의 학습 방법

- 사전 학습(Pre-training)

입력 텍스트의 다음 단어를 예측하는 훈련

입력에 따른 정답 데이터 셋을 따로 구축하지 않아도 됨

- 파인 튜닝(Fine-tuning)

사전 학습된 모델을 특정 작업에 맞게 재학습하는 과정

조건부 확률을 최대화 하도록 학습, 동일한 모델로 다양한 기능을 수행 및 범용성 향상
